## 3. Procédure de reproductibilité (preuve tier-party)

**Objectif** : un avocat, un journaliste, un régulateur, un statisticien indépendant peut **vérifier les chiffres en moins de 30 minutes** sans contacter NAVLYS.

### Étapes
```bash
# 1. Installer Python 3.10+ (gratuit, multi-plateforme)
# 2. Installer les dépendances (toutes gratuites, open source)
pip install yfinance ccxt numpy pandas scipy openpyxl

# 3. Télécharger les fichiers Cartographe (URL navlys.com/lab/m2)
#    - _CARTOGRAPHE_M2_BACKTEST_REEL_PACKAGE_NAVLYS.py
#    - _CARTOGRAPHE_M2_iterations_optim.py

# 4. Vérifier le SHA-256 du script (intégrité)
sha256sum _CARTOGRAPHE_M2_BACKTEST_REEL_PACKAGE_NAVLYS.py
# attendu : 42b10babbe95f1fe58ad3215ef5f366c0c42d31f5a8a9c7837f164bfda45c956

# 5. Lancer
python _CARTOGRAPHE_M2_BACKTEST_REEL_PACKAGE_NAVLYS.py
python _CARTOGRAPHE_M2_iterations_optim.py

# 6. Comparer les chiffres obtenus à ceux publiés
diff _CARTOGRAPHE_M2_METRICS.json     ./reference/_CARTOGRAPHE_M2_METRICS.json
```

**Tolérance acceptable** : ± 0,5 % sur les métriques principales (Sharpe, rendement, DD). Cette tolérance vient :
- des rares révisions de bougies historiques chez Yahoo (réorganisations corporate actions),
- des variations marginales des ordres de pagination CCXT.

Toute déviation > 1 % nécessite investigation et la responsabilité incombe à celui qui modifie le script ou les données. NAVLYS publie la version *figée* du script avec hash SHA-256.
