## 6. Règles de cohérence inter-produits

1. **Un compte Supabase = un utilisateur unique** (table `auth.users`). NAVLYS et NAVBIO partagent la table `users` + tables business distinctes (cf. `04_CODE_INTEGRATION_QUOTAS_CREDITS.md`).
2. **Le statut abonné NAVLYS** est vérifié par middleware avant tout achat NAVBIO. Si actif → application discount −40 % via coupon Stripe `NAVLYS_40` automatique.
3. **L'option NEXT GEN APP** active automatiquement les tables `user_voice_clones` et `user_avatar_videos` (Supabase) + crée la clé ElevenLabs + HeyGen via webhook `checkout.session.completed`.
4. **Crédits supplémentaires** : tables `user_quotas` + `user_credits` (cf. `02_SYSTEME_QUOTAS_CREDITS.md`). Reset mensuel, validité 12 mois glissants.
5. **Annulation NAVLYS** : les achats NAVBIO existants restent acquis à vie. Seul le discount −40 % futur disparaît.
6. **Remboursement NAVBIO** : pro rata des crédits non consommés sous 14 jours uniquement (CGV NAVBIO §X — à valider notaire).
