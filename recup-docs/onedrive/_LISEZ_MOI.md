# 📥 NAVLYS — INBOX DU QG (point de dépôt pour intégration)

> Le QG (manager en chef) ne peut pas lire une autre conversation directement.
> Pour qu'une conversation (ex. **« Nash NAVLYS »**) soit **intégrée au projet**, son contenu doit **arriver ici**, en fichier.

## 🔌 Comment faire entrer une conversation dans le projet (2 façons)

**Option A — la conversation écrit ici (recommandé).**
Dans la conversation à intégrer, colle ceci :
```
⚓ NAVLYS DISPATCH — dépose ton travail au QG.
Écris un fichier dans : OneDrive Downloads/_NAVLYS_INBOX/
Nom : <sujet>-<date>.md  (ex. nash-navlys-2026-05-25.md)
Contenu : un RÉSUMÉ clair (objectif, ce qui a été produit, fichiers/chemins,
décisions, prochaine action) + colle les éléments importants en texte.
But : que le QG l'intègre à _NAVLYS_MASTER_INDEX.md et au bon département.
```
Puis dis-moi au QG : « intègre l'inbox » → je consolide.

**Option B — tu colles le résumé ici.**
Copie-colle dans le QG le résumé / les passages clés de la conversation. Je les intègre tout de suite (index + département concerné).

## 🗂️ Ce que le QG fait à la réception
1. Lit le dépôt. 2. Le range dans le bon département (`_NAVLYS_DEPARTEMENTS/`) ou produit/`NAVBIO/`. 3. Met à jour `_NAVLYS_MASTER_INDEX.md`. 4. Note la décision/action dans `_NAVLYS_QG_COORDINATION.md`.
