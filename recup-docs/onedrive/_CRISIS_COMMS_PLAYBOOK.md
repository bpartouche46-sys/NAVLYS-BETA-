## 🎯 Principes universels (avant tout sous-cas)

1. **Ne jamais nier un fait vérifiable.** Reconnaître + recadrer.
2. **Ne jamais promettre ce qu'on ne peut tenir.** Préférer "je vais vérifier" à "c'est faux".
3. **Toujours signer.** `— BM` ou `— Équipe NAVLYS`.
4. **Toujours rappeler le statut éditorial** dans les sujets sensibles (pas CIF/ORIAS = volontaire).
5. **Ne jamais s'engager sur un délai** sans pouvoir le tenir.
6. **Ne jamais répondre à une attaque par une attaque.** Le silence ou la pédagogie gagnent.

4. **Si demande institutionnelle officielle** : répondre uniquement par voie email, après revue Bruno + (si nécessaire) conseil juridique externe.

## § 2 — Attaque concurrentielle

### Symptômes
- Concurrent finance / influenceur qui mentionne NAVLYS négativement.
- Faux compte qui propage rumeurs ("NAVLYS = arnaque" / "Bruno est CIF déchu").
- Comparaison agressive ("Mon truc à moi est mieux que NAVLYS").

### Action immédiate (sous 4 h)
1. **Capturer** la critique (screenshot + URL).
2. **Évaluer la portée** : <100 vues → ignorer · 100-10k vues → réponse pédagogique · >10k vues ou virale → escalade.
3. **Ne JAMAIS répondre par une attaque.** La règle est : pédagogie + chiffres + signature.

### Réponse publique type (si <10k vues, optionnel)
```
Bonjour [prénom],

Merci de mentionner NAVLYS. Pour clarifier les points soulevés :

1. NAVLYS est un éditeur de contenu pédagogique. Pas un conseiller régulé. Pas un signaleur trading. Pas un vendeur de formation. C'est dans le press kit public.

2. Le Laboratoire NEXT GEN publie autant d'invalidations que de validations. Code Python ouvert. À ce jour : 5 INVAL + 1 VAL partielle. Vérifiable.

3. Tarif : 39 €/mois verrouillé BETA, 49 €/mois ensuite. Pas de surprise. Pas d'upsell.

Si tu veux qu'on en discute en visio publique 20 min : je dis oui. presse@navlys.com.

— BM
```

### Réponse virale (>10k vues) — silence pédagogique
- Ne pas répondre sous le post viral.
- Publier en parallèle un **post propre** sur ses propres réseaux qui répète les 3 points sans nommer l'attaquant.
- Laisser les abonnés se rendre compte par eux-mêmes.

### Faux compte / diffamation
1. Signaler à la plateforme (LinkedIn / X / Insta) avec preuves.
2. Si propos diffamatoires (loi 1881 art. 29) → mise en demeure via avocat.
3. Surtout : **maintenir la cadence de publication propre** — l'algo dilue le négatif.

## § 3 — Bug technique site / app (transparence)

### Symptômes
- navlys.com down ou erreur 500.
- Gate qui ne s'ouvre pas alors qu'il devrait.
- Bug paiement Stripe.
- Email confirmation BETA non envoyé.

### Action immédiate (sous 30 min)
1. **Confirmer le bug** : vérifier soi-même + status.vercel.com.
2. **Publier un message court** sur X / Bluesky / Insta Story.
3. **Pinguer Cowork** pour analyser logs Vercel.
4. **NE PAS retirer le post buggé** — corriger en commentaire.

### Tweet / post crise technique type
```
⚠️ Bref afflux sur navlys.com — on reconnecte la coque.

brunopartouche.com et navbiolife.com fonctionnent en attendant.

Mise à jour dans [X] min.

— Équipe NAVLYS
```

### Update résolution (15-60 min après)
```
✅ navlys.com est de retour.

Cause : [pic de trafic / certif SSL / build concurrent / autre].
Durée incident : [X] min.
Impact : aucun paiement perdu, aucune inscription perdue.

Merci pour la patience.

— Équipe NAVLYS
```

### Postmortem (24-48 h après)
Publier un **billet blog public** type incident report :
- Heure de début / fin
- Cause racine
- Actions correctrices
- Mesures préventives

Pas de blame. Transparence totale. C'est NAVLYS — l'honnêteté est notre fonctionnalité principale.

## § 4 — Bad review virale

### Symptômes
- Post LinkedIn / X / blog : *« J'ai testé NAVLYS pendant X jours, voilà mon avis [négatif]. »*
- Avis Trustpilot / Google négatif.
- Thread Reddit / Discord critique.

### Action immédiate (sous 24 h)
1. **Lire COMPLÈTEMENT** la critique. Pas le titre. La totalité.
2. **Trier** : critique sur le fond (méthode) vs forme (UX) vs malentendu.
3. **Répondre dans le commentaire ou sur le post** — sous 24 h, publiquement, calmement.

### Réponse type (critique de fond — méthode 90/10)
```
Bonjour [prénom],

Merci d'avoir testé NAVLYS et d'avoir pris le temps de partager. Sur le fond :

Tu dis que le 90/10 est trop conservateur / trop agressif / trop lent. Je l'entends. Le 90/10 est un CADRE, pas une recette. Il convient à des profils précis (Tortue / Saumon / Albatros — voir nos 7 profils).

Si tu veux un autre cadre (60/40 par exemple), il y a 1000 ressources gratuites qui l'expliquent mieux que moi. Je ne prétends pas avoir LA méthode universelle.

Ce que je revendique : un cadre QUE JE SUIS depuis 30 ans. Et un Lab qui publie ses échecs.

Si tu veux qu'on en discute 20 min en visio publique : je dis oui.

— BM

🧪 Lab NEXT GEN · éducation seule · pas de conseil personnalisé.
```

### Réponse type (critique de forme — UX, bug, prix)
```
Bonjour [prénom],

Merci pour ce retour précis. Je le note.

→ [Point UX] sera traité dans la prochaine mise à jour (semaine [X]).
→ Sur le prix : 39 €/mois verrouillé BETA reste valable jusqu'au 30 juin. Pas de surprise.
→ Si tu veux qu'on en discute directement : presse@navlys.com.

```

### Critique de malentendu
Recadrer **factuellement** + lien vers FAQ ou press kit. Pas de longueur. Pas d'agressivité.

## § 5 — Question journaliste embarrassante

### Symptômes
- Mail journaliste : *« Vous dites pas CIF — mais avez-vous été refusé à l'examen ? »*
- DM : *« Pourquoi avoir quitté Microsoft / Sela Logistics ? »*
- Question piège en interview : *« Que pensez-vous de [concurrent] ? »* ou *« Combien gagnez-vous ? »*

### Règle d'or
**Anticiper. Préparer. Pas improviser.**

Quand un journaliste appelle, tu n'as **PAS l'obligation de répondre tout de suite**. Tu as le droit de dire :
> *« Bonne question. Je préfère prendre 24 h pour vous répondre proprement par écrit. »*

C'est PROFESSIONNEL. Pas évasif. Pas suspect.

### Réponses types préparées

#### Q : « Vous n'êtes pas CIF — pourquoi ? Vous avez été refusé ? »
**R** :
> *« Non, je n'ai jamais passé l'examen CIF. C'est un choix volontaire et public. Le statut CIF impose le conseil personnalisé — c'est ce que j'ai refusé dès le début. Je tiens à rester éditeur de contenu pédagogique, comme un journaliste finance ou un YouTubeur spécialisé. Cela me préserve la liberté de publier les invalidations sans conflit d'intérêt. C'est dans le press kit page 2. »*

#### Q : « Pourquoi avoir quitté Microsoft / Sela Logistics ? »
**R** :
> *« Je n'ai pas quitté Sela Logistics — j'y suis encore salarié. Microsoft date de mes débuts. Mon parcours en finance, assurance, marketing s'est construit en parallèle pendant 30 ans. NAVLYS est l'aboutissement, pas une rupture. C'est aussi dans le press kit. »*

#### Q : « Que pensez-vous de [concurrent X] ? »
**R** :
> *« Je préfère parler de mon travail. Le concurrent X a sa cible. NAVLYS a la sienne. Je ne fais pas de comparatif public — ce n'est pas le ton du média éducatif que je porte. »*

#### Q : « Combien gagnez-vous ? Quel est votre patrimoine ? »
**R** :
> *« Je suis salarié à temps partiel chez Sela Logistics, et NAVLYS génère ses premiers revenus en BETA depuis le 1er juin 2026. Mon patrimoine personnel n'est pas pertinent pour discuter NAVLYS — c'est précisément parce que NAVLYS est un média éducatif et non un service de gestion. Je publie un revenue report annuel à partir de juin 2027. »*

#### Q : « Et si quelqu'un perd de l'argent en suivant NAVLYS ? »
**R** :
> *« NAVLYS n'a jamais dit à personne quoi acheter. Notre Laboratoire publie autant d'invalidations que de validations. Notre simulateur affiche 3 scénarios honnêtes, dont le pire 10 % historique. Notre disclaimer est en pied de chaque communication : "performances passées ≠ futures, pas de conseil personnalisé, décision personnelle". Si quelqu'un suit aveuglément une publication éducative et perd de l'argent, c'est tragique — mais c'est aussi pourquoi nous insistons tellement sur la pédagogie de la décision. »*

#### Q : « Êtes-vous lié à [secte / parti politique / groupe X] ? »
**R** :
> *« Non. NAVLYS est indépendant. Je suis trompettiste, je pratique des médecines douces (reiki, bols tibétains, acupressure, réflexologie) — ce sont des disciplines personnelles, pas des affiliations à des écoles. Mon parcours est public sur brunopartouche.com. »*

### Quand refuser l'interview
- Si le ton de l'invitation est déjà accusatoire (« Vous prétendez… »).
- Si l'audience est explicitement complotiste ou haineuse.
- Si le délai donné est < 6 h (suggère piège éditorial).
- Si refus de partager les questions à l'avance.

**Refus type** :
```
Bonjour [prénom],

Merci pour votre invitation. Je décline pour cette fois — j'ai besoin de temps de réponse écrit pour bien servir vos lecteurs.

Si une autre format vous convient (réponses écrites, podcast posé, interview avec questions partagées en amont), je dis oui volontiers : presse@navlys.com.

Bien cordialement,
Bruno Mark Partouche
```

## § 6 — Crise multi-axes (combo)

Si plusieurs sous-cas se déclenchent simultanément (ex : bug technique + bad review + journaliste en attente) :

### Ordre de priorité
1. **Sécurité utilisateur** (bug paiement, fuite données) → P0
2. **Sécurité juridique** (régulateur, citation directe officielle) → P0
3. **Réputation virale** (>50k vues) → P1
4. **Bug technique partiel** → P1
5. **Bad review modérée** → P2
6. **Question journaliste hors délai serré** → P2

### Pas de réponse simultanée à tout
- Ne pas répondre à 6 fronts en même temps.
- Centraliser tout dans une **fenêtre 24 h de gestion crise** avec planning des réponses.
- Si > 3 axes simultanés : envisager **24 h de silence intentionnel** sur les réseaux, sauf bug technique critique.
